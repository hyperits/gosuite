# AGENTS

## 项目总览

`gosuite` 是一个 Go 工具库，模块名为 `github.com/hyperits/gosuite`，用于服务端或 Web Go 项目的常用基础能力封装。

| 目录 | 说明 |
|------|------|
| `db/` | 数据库客户端接口与 MySQL、PostgreSQL、Redis 实现 |
| `errors/` | 通用错误变量、错误包装和结构化操作错误 |
| `kit/` | 命令执行、类型转换、运行时调试等工具 |
| `logger/` | 基于 zerolog 的日志封装，支持文件轮转 |
| `net/` | HTTP、邮件、短信相关接口和客户端封装 |
| `providers/` | 第三方服务实现，如阿里云短信、SMTP 邮件 |
| `security/` | 验证码、密码哈希、数字验证码等安全能力 |
| `storage/` | S3 兼容对象存储客户端 |

## 开发原则

- 保持库代码轻量、清晰、可复用，不引入与具体业务强绑定的逻辑。
- 新增能力优先放入语义明确的子包；避免把不相关工具集中塞进已有包。
- 公共 API 改动要谨慎，尽量保持参数和返回值语义稳定。
- 新增第三方依赖前先确认是否必要，避免为了很小的功能引入重依赖。
- 错误信息应便于调用方定位问题，优先复用 `errors/` 中已有错误和包装方式。

## Go 约定

- 当前模块使用 Go `1.20`。
- 提交前运行 `gofmt`；如本地有 `goimports`，同步整理 imports。
- 包名使用简短小写单词，和目录职责保持一致。
- 导出的类型、函数、变量需要有清晰注释，符合 Go lint 文档习惯。
- 测试文件与源码放在同包目录下，文件名使用 `*_test.go`。
- 涉及外部服务的代码应通过接口或配置保持可测试性，避免在单元测试中强依赖真实服务。

## 常用命令

```bash
go test ./...
go vet ./...
gofmt -w .
```

如需整理 imports：

```bash
goimports -w .
```

## 修改建议

- 修改数据库相关逻辑时，分别检查 `db/db.go` 中的公共接口和具体实现包是否一致。
- 修改网络、短信、邮件、存储等 provider 时，确认对应 `net/` 或 `storage/` 中的接口契约没有被破坏。
- 修改 `logger/`、`errors/`、`kit/` 这类基础包时，优先补充或更新测试，因为它们可能被多个包复用。
- 对 Redis、MySQL、PostgreSQL、S3、SMTP、阿里云等需要真实环境的逻辑，单元测试应避免写死生产配置。

## 提交规范

提交信息遵守 README 中的类型前缀：

- `feat` 添加新特性
- `fix` 修复问题
- `style` 无逻辑改动的代码风格调整
- `perf` 性能或优化
- `refactor` 重构
- `revert` 回滚提交
- `test` 测试
- `docs` 文档
- `chore` 依赖或脚手架调整
- `workflow` 工作流优化
- `ci` 持续集成
- `types` 类型定义
- `wip` 开发中
